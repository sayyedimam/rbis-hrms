The system is designed for a localized deployment with future cloud scalability.

[Angular Frontend] -> [FastAPI Backend] -> [Microsoft SQL Server]

Key Decisions

  • Database: Microsoft SQL Server (via SSMS setup).
  • Deployment: Hosted on Local Office PC (currently), Cloud-ready (Future).
  • EmpID Format: Padded string format (e.g., "0001", "0046") preserved across all tables.

üîê Refined Authentication & Access

Custom 3-Field Login: To enhance identity verification, users will log in using:
  1. EmpID (e.g., 0001)
  2. Username
  3. Password

Enrollment Flow

  • Employee Sign-Up: Employees can register themselves. `EmpID` is a mandatory field.
  • Validation: The system checks if the `EmpID` exists and hasn't been registered yet.
  • Role Management: Super Admin creates and authorizes HR Users.

üóÑÔ∏è Database Schema (MySQL)

1. Employees Table

  • id (PK, Auto-inc)
  • emp_id (VARCHAR, Unique) - e.g., "0001"
  • username (VARCHAR)
  • password_hash (VARCHAR)
  • role (ENUM: 'ADMIN', 'HR', 'EMPLOYEE')
  • status (ENUM: 'ACTIVE', 'INACTIVE')

2. Attendance Table

  • id (PK)
  • emp_id (FK)
  • date (DATE)
  • first_in, last_out (TIME)
  • total_duration (DECIMAL/INTERVAL)
  • is_manually_corrected (BOOLEAN)

üõ°Ô∏è Edge Cases & Future Extensions

  • Record Overlap: If an admin uploads a file containing dates already in the DB, the system prompts for "Overwrite" or "Skip".
  • Audit Logs: Every manual correction by a Super Admin tracks the "Original Value", "New Value", and "Timestamp" for audit readiness.
  • Remote Access: Since it's on a local PC, the future cloud deployment will require a migration to an RDS-style MySQL server.
